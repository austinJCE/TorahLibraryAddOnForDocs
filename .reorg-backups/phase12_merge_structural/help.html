User Guide

The add-on now has one main flow: open the right workspace, search or select what you need, confirm it in Preview, then insert it into your Google Doc. Preferences set your defaults. Texts and Voices are where you work. More Options lets you override certain text details for the current session.

Extensions menu Texts Voices Preview More Options Insert

At a glance: what the user workflow should be now

1. Open Use Extensions → [Add-on name] → Texts for standard texts or Voices for source sheets.
2. Search Enter a reference, title, phrase, or Sefaria URL and run the search.
3. Select Choose a result from the list so the sidebar can load the correct preview and available options.
4. Review Confirm language layout, display layout, translation choice, and preview before inserting.
5. Override Open More Options only if you want to change citation, transliteration, niqqud, or cantillation for this session.
6. Insert Click the insert action once the preview matches what you want in the document.
Think of the product this way: Preferences define your defaults, the sidebar shows your live working state, and Preview shows what will be inserted right now.

How to use the Extensions menu

Texts

Open the sidebar directly to the Texts workspace for searching, previewing, and inserting standard Sefaria texts.

Voices

Open the sidebar directly to the Voices workspace for searching source sheets.

Quick Actions

Use document-level shortcuts such as Transform Divine Names and Link Texts with Sefaria without first opening the sidebar.

Preferences

Set your saved defaults and enable experimental features here.

If experimental features are enabled in Preferences, the Extensions menu may also show experimental entry points and extra AI quick actions.

Texts: the standard text workflow

  1. Open Texts.
  2. Type a reference, title, phrase, or paste a Sefaria URL.
  3. Click Search.
  4. Select the result you want from the result list.
  5. Adjust the always-available text controls in the sidebar:
    • Language layout
    • Display layout
    • Translation selection, when more than one translation exists
  6. Review the Preview.
  7. Click the insert action once Preview matches what you want.
The user should not have to open Preferences for ordinary text work. Texts is the normal day-to-day workspace.

Voices: the source-sheet workflow

  1. Open Voices.
  2. Search for the sheet or source-sheet content you want.
  3. Select the result and review the available preview or metadata.
  4. Insert or use the relevant Voices action.

Voices is intentionally narrower than Texts. Text-specific controls should stay out of the way here so the source-sheet workflow remains focused.

What should always stay visible in Texts

Language layout

This is a primary live control and should be available directly in the sidebar.

Display layout

Users should be able to switch layouts without leaving Texts.

Translation selection

When more than one translation exists, the user should be able to select among them directly in the sidebar.

Preview

Preview should always reflect the current live state before insertion.

What belongs in More Options

More Options is for current-session overrides inside Texts. It is not a separate advanced mode. Here you can temporarily change linked source title, source citation, transliteration, vowels, and cantillation without changing saved defaults unless you choose to save them.

Source citation

Turn the source citation block below the inserted text on or off for the current session.

Transliteration

Include transliteration when needed without changing your permanent defaults unless you choose to save them.

Niqqud

Match the Hebrew display details you want to insert.

Cantillation

Use the interactive viewer controls and confirm the result in Preview before inserting.

Changes in More Options should update Preview immediately. They affect the current session unless you explicitly save them as new defaults.

Preferences vs. current session

Preferences

Your per-user saved defaults. These populate Texts when you start a session.

Current session

Your live choices in the sidebar right now. Preview follows these, not just your saved defaults.

Save as defaults

Use this only when you want your current session choices to become the new starting defaults.

Reset to defaults

Return the current session to your saved Preferences.

How result filtering works

After you search, the result list becomes a temporary workspace for narrowing the current result set.

  • Filter by translation language where available.
  • Hide corpus groups from the current results if they are not relevant.
  • Restore hidden corpus groups as needed.
These changes are temporary and last only until the next search.

Preview and versions

  • Preview should show what will be inserted using the current live settings.
  • If multiple translations exist, translation selection should be available in the sidebar.
  • Version-specific details should appear only when relevant and enabled.
  • Open in Sefaria should follow the current display mode whenever possible.

Session Library / History

The history button sits to the left of the search bar. Use it to reopen recent selections and pinned items.

  • Pin a result to keep it in Session Library.
  • Recent items and pinned items are grouped in the history modal.
  • Pinned items sort above recent items within their grouping.

Experimental features
