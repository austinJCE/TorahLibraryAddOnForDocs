Torah Library Add-on for Docs — Release Notes

This release line reflects the current sidebar model and cleanup direction before the shared-include refactor.

Current workflow model

  • Texts and Voices: the top-level sidebar flow is now split by user intent rather than legacy Basic/Advanced naming.
  • Session Settings in More Options: session-level overrides now belong inside Texts → More Options instead of living as a separate top-level mode concept.
  • Interactive Hebrew preview: vowels, cantillation, transliteration, and status now update from a single preview UI.
  • Session Library / History: recent and pinned items are tied directly to current sidebar use and surfaced from the history control near search.

Recent UX and stability work

  • Unified sidebar controls: display and layout use single selector buttons with more stable menu positioning.
  • Results refinement: translation-language filtering, sort, corpus hiding, and restore controls were normalized into one results workflow.
  • Versions cleanup: translation and original versions are treated as separate accordions, with visibility driven by actual availability.
  • Preferences alignment: defaults and one-off session behavior are more clearly separated between Preferences and More Options.
  • Deprecated citation path removal: legacy “Include source citation” behavior is being retired in favor of Translation details.

Refactor-prep status

  • Help, support, and menu copy are being aligned to the Texts / Voices model.
  • Code.gs is being updated so legacy Basic/Advanced entry points route into the new workflow without breaking older hooks.
  • Selector and JS contract fixtures are being tuned before they become hard CI gates.
  • The next major step is extracting shared UI infrastructure while preserving current behavior.
