Torah Library Add-on for Docs — Release Notes

This release continues the 2.0 fork stream with a stronger focus on coherent front-end behavior, clearer working-state controls, and more understandable internal help. The goal is not just to add features, but to make the add-on behave more consistently while people are actually using it inside Google Docs.

What changed in this phase

  • Internal documentation refresh: Support, User Guide, and Release Notes content were updated to reflect the intended modern workflow more clearly.
  • Clearer workspace framing: documentation now distinguishes more explicitly between Texts, Voices, Preferences, and session-level overrides.
  • Better explanation of live state: the docs now explain the difference between saved defaults, current sidebar state, and Preview.
  • Stronger support guidance: bug-report instructions now ask for the workspace, current settings, expected behavior, actual behavior, and whether refresh or reopen changed the result.
  • More explicit expectation-setting: the docs now acknowledge stale sidebar state, session-specific overrides, and workflow-context differences instead of treating them as user error.

Major updates already present in the fork

  • Unified sidebar workflow: a more coherent Find, Preview, and Insert flow for reference lookup, title lookup, and content search results.
  • Translation selector overhaul: normalized and grouped translation entries, improved filtering, and cleaner option labeling.
  • Layout and display controls: stronger output mode and bilingual layout controls with clearer actionability.
  • Structural-node handling: non-insertable structural nodes are detected and messaged clearly before insertion.
  • Hebrew orthography controls: niqqud and ta'amim controls remain available and are more consistently explained.
  • Linking improvements: document linking uses Sefaria's find-refs pipeline with async polling instead of relying only on local regex matching.
  • Preferences and typography additions: persistent Hebrew and translation font and size settings with improved save ergonomics.
  • Divine-name workflow updates: preference-driven Hebrew and English divine-name replacement and document-level transforms.

Focused refinements

  • Improved Hebrew-script reference normalization in unified lookup, including quotes, geresh or gershayim variants, colon or dash variants, and spacing cleanup.
  • Translation option cleanup to remove rendering artifacts such as stray prefixes or leading whitespace.
  • Preferences layout simplified to a single-scroll form with a persistent bottom save action bar.
  • Legacy or experimental features are framed more clearly so they do not compete with the primary workflow unless enabled.

Version 2.0 context

Version 2.0 marked the largest overhaul to the codebase in many years. It moved the project away from accumulated patchwork toward a more maintainable interface and insertion pipeline. That shift also means some behaviors are now stricter, more explicit, and easier to reason about than in earlier versions.

  • Inserted content no longer carries the old visible HTML-tag artifacts that users reported in earlier generations of the add-on.
  • Formatting is preserved more faithfully during insertion, improving the display of bold and italic-rich sources.
  • Reference numbering is based on API response data rather than fragile input assumptions, avoiding older edge cases such as missing or invalid numbering.
  • Preferences were overhauled to support stronger divine-name and orthography defaults.
  • Translation selection became available at preview time when multiple versions exist.
  • Search behavior improved with stronger ranking and matching options.
  • Inserted titles can preserve user-entered form instead of always forcing normalized labels.

What users should know

  • Preferences define how the add-on starts for you.
  • The sidebar reflects your current working state in this session.
  • Preview should show what will be inserted right now.
  • If behavior seems wrong immediately after an update, refresh the document and reopen the add-on before assuming a setting itself is broken.

Deferred backlog

  • Misspelling-tolerant Hebrew reference matching remains deferred until after current linker stability work.

Questions, bug reports, and suggestions

For support, feature requests, or reproducible bug reports, email help@the-merkaz.org or open an issue on GitHub.

The most useful reports include the workspace you were using, what you searched for, the settings you had enabled, what you expected, what happened instead, and whether reopening the add-on or refreshing the document changed the outcome.
